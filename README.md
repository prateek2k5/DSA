# 🚀 45 Days of DSA in C++

A structured collection of **Data Structures and Algorithms problems solved in C++** as part of my **45 Days of DSA journey**.

This repository focuses on learning DSA through **pattern-based problem solving**, understanding the logic behind different approaches, and improving problem-solving skills through consistent practice.

## 📌 About This Repository

Instead of randomly solving problems, I followed a **pattern-based approach to DSA**.

Each section of this repository focuses on a specific problem-solving pattern and contains relevant problems practiced from platforms such as **LeetCode** and **GeeksforGeeks**.

The main objective was not to memorize solutions, but to understand:

* How to identify patterns in a problem
* When to apply a specific data structure or algorithm
* How to move from a brute-force approach to an optimized solution
* How to analyze time and space complexity
* How similar logic can be reused across different problems

## 🧠 My Learning Approach

6. Revise previously solved problems.

## 🔍 Problem-Solving Philosophy

Throughout this journey, I focused on one important idea:

> **DSA is not about memorizing hundreds of solutions. It is about recognizing patterns and understanding how to apply them.**

Many problems that initially appear different often share the same underlying approach.

For example:

* Next Greater Element and Daily Temperatures use the **Monotonic Stack pattern**.
* Maximum Sum Subarray variations are based on **Kadane's Algorithm**.
* Interval-based problems often become simpler after **sorting and comparing overlapping ranges**.
* Frequency-based string problems can often be efficiently solved using **HashMaps**.

Recognizing these connections helped me approach new problems with a more structured mindset.
